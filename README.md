# TrendScanner AI-Based Ecommerce Trend Analyzer

> An AI-assisted multi-agent market intelligence system for analyzing e-commerce product datasets, identifying brand dominance, pricing patterns, popular product features, and potential market opportunities.

## Overview

TrendScanner AI is a modular market analytics platform designed to automate the analysis of structured e-commerce product data.

The system accepts product datasets in CSV format, validates and preprocesses the data, and executes four specialized analytical agents:

* **Brand Intelligence Agent** вҖ” analyzes brand distribution and identifies dominant brands.
* **Pricing Intelligence Agent** вҖ” performs statistical price analysis and identifies representative market price ranges.
* **Feature Intelligence Agent** вҖ” extracts product features and analyzes their frequency and prevalence.
* **Market Gap Agent** вҖ” identifies underrepresented brand-feature combinations using expected-versus-observed frequency analysis.

The outputs generated by these agents are aggregated by an orchestrator and optionally passed to Google Gemini, which converts the structured analytical results into a human-readable executive market summary.

The results are presented through an interactive Streamlit dashboard with visualizations, report generation, and email delivery support.

## Key Features

### Multi-Agent Analytical Architecture

TrendScanner AI separates market analysis into four independent analytical modules.

| Agent         | Responsibility                                             |
| ------------- | ---------------------------------------------------------- |
| Brand Agent   | Brand frequency, ranking, and market presence analysis     |
| Pricing Agent | Descriptive statistics and quartile-based pricing analysis |
| Feature Agent | Feature extraction, normalization, and frequency analysis  |
| Gap Agent     | Expected-versus-observed market gap detection              |

### Data Validation and Preprocessing

The system performs preprocessing before executing the analytical agents.

Supported operations include:

* CSV file validation
* Required column checking
* Missing value handling
* Duplicate removal
* Data type conversion
* Configurable column mapping
* Dataset summary generation

### Brand Intelligence

The Brand Agent performs frequency-based market analysis.

For every brand:

```text
Brand Confidence = Brand Product Count / Total Number of Records
```

This helps identify dominant brands and understand their relative presence within the uploaded dataset.

### Pricing Intelligence

The Pricing Agent performs descriptive statistical analysis of product prices.

The following metrics are calculated:

* Minimum Price
* Maximum Price
* Mean Price
* Median Price
* Standard Deviation
* First Quartile (Q1)
* Third Quartile (Q3)

The system represents the central market price range using:

```text
Optimal Price Range = [Q1, Q3]
```

Optional price enrichment functionality is also available for processing market prices and currency-related transformations.

### Feature Intelligence

Product feature strings are normalized and tokenized before analysis.

Supported delimiters include:

```text
,
;
|
```

For example:

```text
5G, OLED, 128GB
```

is transformed into individual features:

```text
5G
OLED
128GB
```

The frequency and prevalence of each feature are then calculated.

### Market Gap Detection

The Gap Agent identifies underrepresented brand-feature combinations.

For a brand `b` and feature `f`, the expected frequency is calculated as:

```text
Expected(b,f) = BrandCount(b) Г— FeatureCount(f) / TotalRecords
```

The Gap Score is calculated as:

```text
GapScore(b,f) = (Observed(b,f) - Expected(b,f)) / Expected(b,f)
```

Interpretation:

```text
Gap Score > 0     вҶ’ Overrepresented combination

Gap Score вүҲ 0     вҶ’ Balanced combination

Gap Score < 0     вҶ’ Underrepresented combination
```

When the Gap Score falls below the configured threshold, the combination is identified as a potential market opportunity.

## AI-Powered Executive Summarization

After all analytical agents complete their execution, their outputs are combined into a structured result.

```text
Brand Results
      +
Pricing Results
      +
Feature Results
      +
Gap Results
      вҶ“
Structured Analytical Output
      вҶ“
Gemini LLM
      вҶ“
Executive Market Summary
```

Google Gemini is used only as the natural-language summarization layer.

The core market calculations are performed deterministically using Python, Pandas, and statistical methods.

This separation reduces dependence on the LLM for analytical correctness and improves the explainability of the system.

## Project Workflow

```text
1. User uploads an e-commerce product CSV dataset.

2. The ingestion module reads the dataset into a Pandas DataFrame.

3. The validation module checks required columns, handles missing values, and removes duplicate records.

4. Optional price enrichment operations are applied.

5. The orchestrator executes the analytical agents.

6. The Brand Agent analyzes brand distribution.

7. The Pricing Agent performs descriptive statistical analysis.

8. The Feature Agent extracts and analyzes product features.

9. The Gap Agent detects underrepresented brand-feature combinations.

10. Agent outputs are aggregated into a structured result.

11. The structured results can be passed to Google Gemini for executive summarization.

12. Results are displayed through the Streamlit dashboard.

13. Reports can be exported and delivered through the reporting utilities.
```

## Input Dataset Format

The minimum required dataset structure is:

| Column    | Description                   |
| --------- | ----------------------------- |
| `brand`   | Product manufacturer or brand |
| `price`   | Product price                 |
| `feature` | Product features              |

Optional columns such as `model` and `category` may also be included.

Example:

```csv
brand,price,feature,model,category
Samsung,24999,"5G,AMOLED,128GB",Galaxy A Series,Smartphone
Apple,79999,"5G,OLED,256GB",iPhone Series,Smartphone
Xiaomi,18999,"5G;AMOLED;Fast Charging",Redmi Note Series,Smartphone
OnePlus,34999,"5G|AMOLED|Fast Charging",Nord Series,Smartphone
```

## Design Principles

TrendScanner AI follows several architectural principles.

**Modularity:** Each analytical agent has a separate responsibility.

**Explainability:** Core market insights are generated using deterministic statistical methods.

**Separation of Concerns:** Data ingestion, preprocessing, analytics, LLM summarization, visualization, and reporting are implemented as separate layers.

**LLM Independence:** Failure of the LLM summarization layer does not prevent the core analytical pipeline from producing results.

**Extensibility:** Additional analytical agents and data sources can be integrated into the existing architecture.

---

## Current Limitations

* Analysis depends on the quality and representativeness of the uploaded dataset.
* The current system primarily supports structured CSV product data.
* The system does not currently implement time-series forecasting.
* No machine learning model is trained as part of the core analytical pipeline.
* Feature extraction is based on delimiter-based tokenization rather than advanced semantic NLP.
* Market-gap detection identifies statistical underrepresentation and does not guarantee commercial demand.
* External API functionality depends on API availability and configured credentials.
